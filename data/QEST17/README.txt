=========================================
README relative to the QEST17 submission
=========================================

The PIMC benchmark presented in this submission is based on the NAND model from PRISM
(cf http://www.prismmodelchecker.org/benchmarks/models.php#dtmcs)
The PIMCs in Text Format has been produced by extracting the transition matrix of the NAND models produced by PRISM.

1) PIMCs in text format used for consistency
2) PIMCs in text format used for reachability properties
3) PIMCs in SVG format for visualization

1) We instantiated the NAND model with N ranging from 2 to 10 producing the PIMCs:
- pimcs/nand_N_2_K_1.pimc
- pimcs/nand_N_3_K_1.pimc
- pimcs/nand_N_5_K_1.pimc
- pimcs/nand_N_10_K_1.pimc

2) Furthermore, for the existential reachability benchmark, we performed some dissimulation and reduction operation while preserving probabilities. The reduced PIMCs are :
- pimcs/nand_N_2_K_1_reach
- pimcs/nand_N_3_K_1_reach
- pimcs/nand_N_5_K_1_reach
- pimcs/nand_N_10_K_1_reach

3) The "pimcs" directory contains the SVG representation of the PIMCs (thanks to our tool):
./src/pimc_modeler -i <PIMC_IN_TXT_FORMAT> -o <PIMC_IN_DOT_FORMAT>.dot -dot

4) Folders "smt_consistency", "smt_reach_qualitative", and "smt_reach_quantitative" resp. contain the CSP in SMT-LIB format generated by our tools from our PIMC benchmark. Command used are:
> ./src/pimc_modeler -i <PIMC_IN_TXT_FORMAT> -o <CSP_IN_SMT_FORMAT>.smt2 -smt -cons
> ./src/pimc_modeler -i <PIMC_IN_TXT_FORMAT> -o <CSP_IN_SMT_FORMAT>.smt2 -smt -qualReach
> ./src/pimc_modeler -i <PIMC_IN_TXT_FORMAT> -o <CSP_IN_SMT_FORMAT>.smt2 -smt -quantReach

5) Use the generated CSP in SMT-LIB format as input of your SMT solver. For instance with Z3:
z3 -smt2  <CSP_IN_SMT_FORMAT>.smt2
